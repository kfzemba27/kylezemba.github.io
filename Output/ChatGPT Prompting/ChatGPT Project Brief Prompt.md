## Tool Card Schema

Every tool entry needs all of these fields filled in:

- **Name**
- **Company**
- **Tagline** — One punchy line. Not a sentence, not a paragraph.
- **What it is** — 2-3 sentences max. Lead with what it enables, not what it technically does.
- **Best For** — Specific team function. "Creative teams concepting campaign visuals" not just "Creatives." Options: Strategy / Creative / Production / Design / Analytics / All
- **Pros** — Sharp, specific advantages tied to real workflow benefits. No generic filler.
- **Cons** — Honest, practical limitations. Pricing barriers, learning curves, output quality gaps, functionality limits.
- **How to use it** — Genuinely actionable tips. What would a first-time user benefit from knowing? Prompt strategies, workflow suggestions, feature callouts.
- **Highlight Features** — The things you'd mention if someone asked "why this one over alternatives?"
- **Pricing** — Specific tiers with dollar amounts where possible.
- **URL**

## Writing Rules — Follow These Strictly

**Do:**
- Lead with what the tool enables, not what it technically does
- Use active voice
- Be specific — name actual advertising and marketing use cases
- Acknowledge trade-offs and limitations honestly
- Write like you're briefing a sharp colleague, not writing a review
- Keep "What it is" to 2-3 sentences, no exceptions
- Frame everything through the lens of advertising and marketing workflows

**Don't:**
- Use em dashes
- Use exclamation points
- Use buzzwords without substance: "revolutionary," "game-changing," "next-gen," "cutting-edge"
- Write marketing copy about the tools
- Use "leverage" as a verb
- Start sentences with "So," or "Basically,"
- Be sycophantic toward any tool or company
- Pad content to fill space

**Tone:** Practitioner, not pundit. Confident but not arrogant. Strategic, not just tactical. Write like someone who has actually used these tools in real work — not someone summarizing press releases.
